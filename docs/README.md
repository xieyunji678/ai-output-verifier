## 1. 项目概述

### 1.1 项目背景

AI Agent 广泛应用的同时，其输出内容的真实性、准确性问题日益突出。AI 幻觉（Hallucination）和事实错误可能导致严重后果，尤其在医疗、法律、金融等专业领域。

### 1.2 产品定位

**AI Output Verifier** 是一个面向企业/个人的 AI 内容验证平台，帮助用户：
- 验证 AI 输出的事实准确性
- 识别 AI 胡编乱造的内容
- 提供可追溯的证据链

### 1.3 目标用户

| 用户类型 | 使用场景 |
|----------|----------|
| 内容创作者 | 审核 AI 辅助生成的文章 |
| 企业客服 | 检查 AI 客服回复的准确性 |
| 开发者 | 调试 RAG 应用，评估输出质量 |
| 研究人员 | 分析 LLM 幻觉模式 |

## 2. 功能需求

### 2.1 核心功能列表

#### F1: 文本输入
- [ ] 支持粘贴/输入待验证文本（最大 10,000 字符）
- [ ] 支持上传 .txt / .md 文件
- [ ] 支持 URL 抓取（可选）

#### F2: 事实核查
- [ ] 自动提取文本中的事实断言（Factual Claims）
- [ ] 对每个断言进行真伪判断
- [ ] 返回验证结果：True / False / Uncertain
- [ ] 提供证据来源链接

#### F3: 幻觉检测
- [ ] 识别文本中不可验证的模糊表述
- [ ] 标记可能为幻觉的内容片段
- [ ] 给出置信度评分（0-100%）

#### F4: 结果展示
- [ ] 可视化标注错误/可疑位置
- [ ] 一键复制修订后的版本
- [ ] 导出验证报告（PDF/Markdown）

#### F5: 验证历史
- [ ] 保存验证记录（仅本地/需登录）
- [ ] 支持历史记录查询

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 |
|------|------|
| 首屏加载 | < 2s |
| 验证响应时间 | < 30s（单次<5000字符） |
| 并发支持 | 100 用户同时在线 |

### 3.2 准确率

- 事实核查准确率：> 85%
- 幻觉检测召回率：> 80%
- （后续通过用户反馈持续优化）

### 3.3 可用性

- 界面简洁，无需培训即可上手
- 移动端适配（响应式布局）
- 错误提示清晰

### 3.4 安全

- 用户输入数据仅用于当次验证
- 可选：本地部署模式（数据不出网）

## 4. 技术方案

### 4.1 技术选型

| 层级 | 推荐技术 |
|------|----------|
| 前端 | Next.js 14 + React + Tailwind CSS |
| 后端 | Python FastAPI |
| AI 模型 | GPT-4o / Claude 3.5（API）或 本地 LLM |
| 知识检索 | Elasticsearch + 向量搜索 |
| 数据库 | PostgreSQL + pgvector |
| 部署 | Vercel + Railway / Docker |

### 4.2 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端 (Next.js)                       │
│   文本输入  │  结果展示  │  历史记录  │  用户中心          │
└────────────────────────────┬────────────────────────────────┘
                             │ REST API
┌────────────────────────────▼────────────────────────────────┐
│                      后端 (FastAPI)                          │
│   /verify  │  /history  │  /user                            │
└─────────┬───────────────────────────────────────┬───────────┘
          │                                       │
          ▼                                       ▼
┌──────────────────────┐               ┌──────────────────────┐
│   验证服务            │               │   知识库服务          │
│   - 断言提取         │               │   - 搜索             │
│   - 置信度评估       │               │   - 向量检索         │
└──────────────────────┘               └──────────────────────┘
          │
          ▼
┌──────────────────────┐
│   LLM 调用           │
│   (OpenAI/Claude)   │
└──────────────────────┘
```

### 4.3 数据流

1. **用户提交文本** → 后端接收
2. **断言提取** → 使用 LLM 从文本中提取可验证的事实断言
3. **并行处理**：
   - 知识检索：查询可信来源
   - 不确定性评估：LLM 自我评估置信度
4. **结果聚合** → 返回带证据的验证结果
5. **前端渲染** → 高亮标注 + 证据展示
